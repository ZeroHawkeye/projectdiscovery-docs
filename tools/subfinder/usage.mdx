## 获取帮助

使用 `subfinder -h` 显示所有帮助选项。

## Subfinder 帮助选项

```
Flags:
INPUT:
  -d, -domain string[]  用于查找子域名的域名
  -dL, -list string     包含用于子域名发现的域名列表的文件

SOURCE:
  -s, -sources string[]           用于发现的特定来源 (-s crtsh,github)。使用 -ls 显示所有可用来源。
  -recursive                      只使用能够递归处理子域名的来源（例如 subdomain.domain.tld 与 domain.tld）
  -all                            使用所有来源进行枚举（速度较慢）
  -es, -exclude-sources string[]  从枚举中排除的来源 (-es alienvault,zoomeyeapi)

FILTER:
  -m, -match string[]   要匹配的子域名或子域名列表（文件或逗号分隔）
  -f, -filter string[]   要过滤的子域名或子域名列表（文件或逗号分隔）

RATE-LIMIT:
  -rl, -rate-limit int  每秒发送的最大 http 请求数
  -rls value            以键值格式设置每个提供商每秒发送的最大 http 请求数 (-rls "hackertarget=10/s,shodan=15/s")
  -t int                用于解析的并发 goroutine 数量（仅限 -active）（默认为 10）

UPDATE:
   -up, -update                 更新 subfinder 到最新版本
   -duc, -disable-update-check  禁用自动 subfinder 更新检查

OUTPUT:
  -o, -output string       输出文件路径
  -oJ, -json               以 JSONL(ines) 格式输出
  -oD, -output-dir string  输出目录（仅限 -dL）
  -cs, -collect-sources    在输出中包含所有来源（仅限 -json）
  -oI, -ip                 在输出中包含主机 IP（仅限 -active）

CONFIGURATION:
  -config string                参数配置文件（默认为 "$CONFIG/subfinder/config.yaml"）
  -pc, -provider-config string  提供商配置文件（默认为 "$CONFIG/subfinder/provider-config.yaml"）
  -r string[]                   用于解析的解析器列表，以逗号分隔
  -rL, -rlist string            包含要使用的解析器列表的文件
  -nW, -active                  仅显示活跃的子域名
  -proxy string                 subfinder 使用的 http 代理
  -ei, -exclude-ip              从域名列表中排除 IP

DEBUG:
  -silent             输出中仅显示子域名
  -version            显示 subfinder 版本
  -v                  显示详细输出
  -nc, -no-color      禁用输出中的颜色
  -ls, -list-sources  列出所有可用来源

OPTIMIZATION:
  -timeout int   等待超时的秒数（默认为 30）
  -max-time int  等待枚举结果的分钟数（默认为 10）
  ```
